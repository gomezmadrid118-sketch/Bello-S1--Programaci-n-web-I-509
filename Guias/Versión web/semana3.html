Unidad I - Generalidades de Javascript

Semana 3: Ciclos

Temas: Ciclos en JavaScript: estructuras for, while y do-while.

Situación: La App de Café

En la sesión anterior, aprendimos a usar if para tomar una decisión. Por ejemplo, podríamos comprobar si la nota de un estudiante es mayor a 3.0 para saber si aprobó.

Ahora, ¿qué pasa si tenemos un salón con 40 estudiantes? ¿Escribimos 40 bloques if diferentes? 

if (estudiante1.nota > 3.0) { ... } 
if (estudiante2.nota > 3.0) { ... } 
... 
if (estudiante40.nota > 3.0) { ... }

Eso es ineficiente, imposible de mantener y viola el principio de "No Te Repitas" (DRY - Don't Repeat Yourself).

Necesitamos una forma de decirle a JavaScript:

"Toma esta lista de estudiantes y repite esta misma lógica de if para cada uno de ellos".

Esa instrucción de "repetir" es un ciclo o bucle. Los bucles son el motor de nuestras aplicaciones; son la maquinaria que nos permite procesar miles de datos con unas pocas líneas de código.

El Concepto de Iteración

Un ciclo (o bucle) es una estructura de control que repite un bloque de código mientras una condición sea verdadera (true).

Tipos de Ciclos

El Ciclo for (La Iteración Definida)

El ciclo for es la estructura de repetición más común. Se usa cuando sabemos de antemano cuántas veces queremos que se repita una acción. (Ej. "Repite esto 10 veces", "Repite esto por cada uno de los 40 estudiantes").

Analogía:

Un ciclo for es como un entrenador que te dice: "Corre 5 vueltas a la pista".

  1. Punto de Partida: Empiezas en la vuelta 0 (o 1).
  2. Condición para Seguir: ¿Has corrido menos de 5 vueltas? Si es sí, sigue corriendo.
  3. Acción por Vuelta: Al final de cada vuelta, anotas 1 vuelta más en tu contador.

Anatomía del Código

El for se define con tres partes clave dentro de los paréntesis, separadas por punto y coma: for (inicialización; condición; incremento) { ... } 

// Este ciclo imprimirá los números del 0 al 4
for (let i = 0; i < 5; i++) {
  console.log('El número es: ' + i);
}

1. let i = 0; (La Inicialización): Es el "Punto de Partida". Creamos una variable (¡usando let!) que actuará como nuestro contador. Esto solo se ejecuta una vez al inicio.

2. i < 5; (La Condición): Es la "Regla para Continuar". Antes de cada vuelta, JavaScript revisa esta condición. Si es true, ejecuta el bloque de código. Si es false, el ciclo se detiene.

3. i++ (El Incremento): Es el "Paso". Se ejecuta después de cada vuelta. i++ (de la Sesión 1) suma 1 a nuestro contador, acercándonos al final.

Profundización

¿Por qué let y no var? Si usáramos var i, esa variable i se "fugaría" del ciclo y existiría en todo nuestro código, pudiendo causar errores. Al usar let i, la variable i solo existe dentro del bloque {...} del ciclo, lo cual es mucho más seguro y predecible.

El Ciclo while

Usamos un while cuando no sabemos cuántas veces necesitamos repetir. La repetición no depende de un contador, sino de que una condición externa siga siendo verdadera.

Ejemplo de la vida real: Mientras el jugador tenga vidas (vidas > 0), sigue jugando. No sabes si el juego durará 1 minuto o 3 horas. El ciclo continúa mientras la condición se cumpla.

¡Peligro! El Bucle Infinito

El while es poderoso pero peligroso. Si la condición nunca se vuelve false, el código se repetirá para siempre, congelando el navegador. Algo dentro del while debe, eventualmente, cambiar la condición.

Anatomía del Código

let vidas = 3;

while (vidas > 0) {
  console.log(`¡Tienes ${vidas} vidas!`);
  
  // Simulamos que el jugador pierde una vida
  // ESTA LÍNEA ES CRUCIAL. Modifica la condición.
  vidas--; // Sin esto, sería un bucle infinito.
}

console.log('Game Over');

Salida:

¡Tienes 3 vidas!
¡Tienes 2 vidas!
¡Tienes 1 vidas!
Game Over

El Ciclo do...while

Es casi idéntico al while, con una sola diferencia crucial: el do...while siempre se ejecuta al menos una vez.

Caso de uso: Un menú de opciones donde primero muestras el menú y luego preguntas si quiere pedir algo del menú.

Se usa en situaciones donde necesitas que el código se ejecute una vez antes de comprobar si debe repetirse.

Anatomía del Código

let contrasena;

do {
  // Este bloque se ejecuta SIEMPRE al menos una vez
  contrasena = prompt('Introduce tu contraseña:'); 
} while (contrasena !== '12345'); // La condición se revisa DESPUÉS

console.log('¡Bienvenido!');

Diferencia Clave: while vs do...while

while: Pregunta PRIMERO

let x = 0;
while (x > 5) {
  console.log(x);
}
// No imprime nada
// porque 0 > 5 es false

do...while: Actúa PRIMERO

let y = 0;
do {
  console.log(y);
} while (y > 5);
// Imprime 0
// Ejecuta antes de
// revisar la condición

El Bucle for...of

Este es el sucesor moderno del ciclo for clásico cuando quieres recorrer un Arreglo. Es mucho más limpio y menos propenso a errores.

Para qué sirve:

Para iterar sobre los valores de un objeto "iterable" (como un Arreglo, un String, un Mapa, etc.).

Casos de Uso

Ciclo for (clásico):

Eres un repartidor al que le dicen: "Ve a la casa #0, luego a la casa #1, luego a la #2...". Te importa mucho el índice (el número de la casa).

Ciclo for...of:

Eres un repartidor al que le dan una lista de clientes. Vas al "cliente A", luego al "cliente B". No te importa si son la casa #0 o la #1 (índice), solo te importa el valor (el cliente).

Anatomía del Código:

const miArray = ['manzana', 'banana', 'naranja'];

// El 'for...of'
for (const fruta of miArray) {
  // 'fruta' toma el VALOR de cada elemento en cada vuelta
  console.log(fruta);
}

// Resultado:
// "manzana"
// "banana"
// "naranja"

Cuándo usarlo:

Es la forma preferida y moderna de recorrer un arreglo cuando solo te importa el valor de cada elemento y no necesitas saber su posición (el índice i).

El Bucle for...in - El Iterador de Propiedades de Objetos

Este bucle se ve similar al for...of, pero es fundamentalmente diferente y es una fuente común de confusión.

Para qué sirve:

Para iterar sobre las propiedades (las "claves" o keys) de un Objeto.

Forma de Pensar:

A este bucle no le importa quién vive en la casa (valor), le importa el nombre de las habitaciones (clave). Si le das un objeto persona, te dirá que tiene una habitación "nombre", una "edad" y una "profesión".

¡Advertencia Importante!

Nunca uses for...in para recorrer un Arreglo. Técnicamente "funciona", pero lo hará de forma incorrecta:

  1. Iterará sobre los índices ('0', '1', '2') como si fueran strings, no sobre los valores.
  2. Puede iterar sobre propiedades inesperadas que no son parte de los datos.

Resumen de la Diferencia:

for...of

Para valores de Arreglos

for...in

Para claves de Objetos

Anatomía del Código: 

const persona = {
  nombre: 'Ana',
  edad: 30,
  profesion: 'Doctora'
};

// El 'for...in'
for (const clave in persona) {
  // 'clave' toma el NOMBRE de la propiedad en cada vuelta
  console.log(clave); // 'nombre', 'edad', 'profesion'
  
  // Para ver el valor, debes usar la clave
  console.log(persona[clave]); // 'Ana', 30, 'Doctora'
}

Mención Honorífica: .forEach() - El Método de Array

No es un "bucle" en el sentido de una sentencia de JavaScript, sino un método que tienen los arreglos. Es la forma "funcional" de hacer un for...of.

Anatomía del Código:

const miArray = ['manzana', 'banana', 'naranja'];

// El .forEach() recibe una función callback
miArray.forEach(function(fruta) {
  console.log(fruta);
});

Salida:

manzana
banana
naranja

Control de Ciclos: break y continue

A veces necesitamos alterar el flujo normal de un ciclo.

break (La Salida de Emergencia)

Termina el ciclo inmediatamente, sin importar si la condición se sigue cumpliendo.

Ejemplo de Uso:

Estoy buscando un nombre en una lista de 1000. Lo encontré en la posición 50. ¡No necesito revisar los 950 restantes! ¡break!

Ejemplo Práctico:

const numeros = [2, 4, 6, 8, 10, 15, 20];
let encontrado = false;

for (let i = 0; i < numeros.length; i++) {
  if (numeros[i] === 15) {
    console.log("¡Encontré el 15 en posición " + i);
    encontrado = true;
    break; // Sale del for inmediatamente
  }
}

console.log("Búsqueda terminada");

Salida:

¡Encontré el 15 en posición 5
Búsqueda terminada

continue (Saltar esta Vuelta)

Detiene la iteración actual y salta directamente a la siguiente (al i++ en un for).

Ejemplo de Uso:

Estoy procesando 100 emails. El email actual es spam. No lo proceso (continue) y salto al siguiente email.

Ejemplo Práctico:

for (let i = 1; i <= 10; i++) {
  // Si es el número 5, lo saltamos
  if (i === 5) {
    continue; // Salta a la siguiente iteración
  }
  console.log("Número: " + i);
}

Salida:

Número: 1
Número: 2
Número: 3
Número: 4
Número: 6
Número: 7
Número: 8
Número: 9
Número: 10

Comparación Rápida:

break

Termina el ciclo completo. Continúa después del ciclo.

continue

Salta esta iteración. El ciclo continúa.

Preguntas Frecuentes

1. ¿Cuál es la diferencia real entre for y while?

Es la pregunta más importante. Usa for cuando tengas una iteración definida (sabes cuántas veces, ej: 10 vueltas, o una vez por cada elemento de una lista). Usa while cuando tengas una iteración indefinida (no sabes cuántas veces, ej: "hasta que el usuario escriba 'salir'", "mientras el jugador tenga vidas").

Resumen: for = conoces el final. while = no conoces el final.

2. Mi navegador se congeló cuando usé un while. ¿Qué pasó?

Creaste un bucle infinito. Esto sucede cuando la condición de tu while empieza siendo true y nada dentro del bucle hace que se vuelva false.

❌ Ejemplo incorrecto (bucle infinito): 

let i = 0;
while (i < 10) {
  console.log(i);
  // ¡La variable i nunca cambia!
  // Siempre será 0, por lo que 0 < 10 siempre es true
}

✅ Solución (incrementa la variable):

let i = 0;
while (i < 10) {
  console.log(i);
  i++; // ¡Incrementa la variable!
}

3. ¿Por qué siempre se usa la variable i?

Es una convención histórica que viene de "Índice" o "Iterador". Si tienes ciclos anidados (uno dentro de otro), se suele usar j, luego k.

Sin embargo, puedes llamarla como quieras (contador, numero, vuelta, etc.), pero i es el estándar universal.

Ejemplo de ciclos anidados (usa i, j): 

// Ciclos anidados: tablas de multiplicar
for (let i = 1; i <= 3; i++) {
  for (let j = 1; j <= 3; j++) {
    console.log(i + " × " + j + " = " + (i * j));
  }
}

Ejemplo 2: El Buscador de Números Pares (while + if)

Vamos a buscar el primer número par mayor a 100 generado aleatoriamente: 

let encontrado = false;
let intentos = 0;

while (!encontrado) {
    // Generamos un número al azar entre 90 y 110
    let numeroRandom = Math.floor(Math.random() * (110 - 90 + 1) + 90);
    intentos++;
    
    console.log(`Intento ${intentos}: Salió el ${numeroRandom}`);

    if (numeroRandom > 100 && numeroRandom % 2 === 0) {
        console.log(`¡Encontrado! El ${numeroRandom} es mayor a 100 y par.`);
        encontrado = true; // Esto detendrá el ciclo
    }
}

Salida (ejemplo, puede variar):

Intento 1: Salió el 95
Intento 2: Salió el 103
Intento 3: Salió el 98
Intento 4: Salió el 102
¡Encontrado! El 102 es mayor a 100 y par.

Conceptos utilizados:

  • while: Repetimos hasta encontrar el número
  • if: Verificamos si es par y mayor a 100
  • Math.random(): Generamos números aleatorios
  • % (módulo): % 2 === 0 verifica si es par
  • encontrado = true: Detiene el ciclo

Práctica Acompañada

Misión

Calcular la suma de todos los números del 1 al 100

(1 + 2 + 3 + ... + 100)

Paso a Paso

1. Abre tu VSC

Visual Studio Code o tu editor de texto favorito

2. Crea una variable "acumuladora"

Necesitamos una "caja" para guardar la suma total. let sumaTotal = 0;

3. Configura el ciclo for

Queremos un ciclo que se ejecute 100 veces, con un contador i que vaya del 1 al 100

4. Escribe la lógica del ciclo

Dentro del for, toma el valor actual de sumaTotal y súmale el valor actual del contador i

5. Muestra el resultado

Después de que el ciclo termine, imprime el valor final de sumaTotal en la consola

Solución Propuesta:

// 1. Crear la variable acumuladora
let sumaTotal = 0;
console.log(`Valor inicial de la suma: ${sumaTotal}`);

// 2. Configurar el ciclo for (del 1 al 100)
// Empezamos i en 1 y la condición es i <= 100
for (let i = 1; i <= 100; i++) {
  
  // 3. Lógica del ciclo
  // En cada vuelta, suma el valor de 'i' al acumulador
  sumaTotal = sumaTotal + i; 
  // O en forma corta: sumaTotal += i;
  
  // Opcional: ver el progreso
  // console.log(`Sumando ${i}, la suma ahora es: ${sumaTotal}`);
}

// 4. Mostrar el resultado final
console.log(`La suma total de los números del 1 al 100 es: ${sumaTotal}`); 
// Resultado esperado: 5050

Resultado esperado:

Valor inicial de la suma: 0
La suma total de los números del 1 al 100 es: 5050

Conceptos Clave:

  • Variable acumuladora: sumaTotal guarda el resultado parcial
  • Ciclo for: Se repite 100 veces automáticamente
  • Operador +=: sumaTotal += i es equivalente a sumaTotal = sumaTotal + i
  • Fórmula matemática: La suma de 1 al 100 es siempre 5050 (n × (n+1) / 2)

Práctica B: Juego de Dados - ¡Evita el 1!

Objetivo del Juego

El objetivo es sacar un 6. Empiezas con 3 vidas. Cada vez que lanzas y no sacas un 6, sigues jugando. Pero si sacas un 1, pierdes una vida.

Condiciones de victoria/derrota:

  • ✅ Ganas si sacas un 6
  • ❌ Pierdes si te quedas sin vidas

Paso a Paso

1️⃣ Crea las variables de estado

  • Crea una variable let vidas y asígnale 3
  • Crea una variable let dado y asígnale 0 (para guardar cada lanzamiento)
  • Crea una variable let ganoElJuego y asígnale false (bandera/flag)

2️⃣ Configura el ciclo while

El juego debe continuar mientras la variable vidas sea mayor que 0

3️⃣ Lanza el dado

Genera un número aleatorio del 1 al 6 y guárdalo en dado

Pista: dado = Math.floor(Math.random() * 6) + 1;

4️⃣ Imprime el resultado

Muestra qué sacaste. Ej: console.log(`Lanzaste un: ${dado}`);

5️⃣ Lógica condicional (if...else if...else)

SI (if) dado === 6:

• Imprime "¡Sacaste un 6! ¡Ganaste!"

• Cambia ganoElJuego = true

• Usa break; para salir

SI NO, PERO SI (else if) dado === 1:

• Resta 1 a vidas: vidas--

• Imprime cuántas vidas quedan

SI NO (else):

• Imprime "Salió un [número]. No pasa nada, vuelves a lanzar."

6️⃣ Muestra el resultado final

Después del while, verifica si perdiste: if (ganoElJuego === false) { console.log('¡Game Over!...'); }

Solución Propuesta:

// 1. Variables de estado
let vidas = 3;
let dado = 0;
let ganoElJuego = false;

// 2. Ciclo while (mientras haya vidas)
while (vidas > 0) {
  
  // 3. Lanza el dado
  dado = Math.floor(Math.random() * 6) + 1;
  
  // 4. Imprime el resultado
  console.log(`🎲 Lanzaste un: ${dado}`);
  
  // 5. Lógica condicional
  if (dado === 6) {
    // ¡Ganaste!
    console.log(`✅ ¡Sacaste un 6! ¡GANASTE EL JUEGO!`);
    ganoElJuego = true;
    break; // Salir inmediatamente
    
  } else if (dado === 1) {
    // Perdiste una vida
    vidas--;
    console.log(`❌ ¡Uy! Salió un 1. Pierdes una vida. Te quedan ${vidas} vidas.`);
    
  } else {
    // No pasa nada
    console.log(`➡️ Salió un ${dado}. No pasa nada, vuelves a lanzar.`);
  }
}

// 6. Resultado final
if (ganoElJuego === false) {
  console.log(`💀 ¡Game Over! Te quedaste sin vidas. Perdiste el juego.`);
}

Salida ejemplo (puede variar):

Lanzaste un: 3
➡️ Salió un 3. No pasa nada, vuelves a lanzar.
🎲 Lanzaste un: 1
❌ ¡Uy! Salió un 1. Pierdes una vida. Te quedan 2 vidas.
🎲 Lanzaste un: 4
➡️ Salió un 4. No pasa nada, vuelves a lanzar.
🎲 Lanzaste un: 1
❌ ¡Uy! Salió un 1. Pierdes una vida. Te quedan 1 vidas.
🎲 Lanzaste un: 6
✅ ¡Sacaste un 6! ¡GANASTE EL JUEGO!

Conceptos Clave Utilizado:

  • Variables de estado: vidas, dado, ganoElJuego
  • Ciclo while: Repetir hasta que vidas sea 0
  • Math.random(): Generar números aleatorios
  • Condicionales if...else if...else: Tomar decisiones según el resultado del dado
  • Break: Salir del ciclo cuando ganas
  • Operador decremento --: vidas-- para perder vidas
  • Flag/Bandera: ganoElJuego para saber si ganaste o perdiste

Troubleshooting: Errores Comunes en Ciclos

El Loop Infinito

Error:

for (let i = 0; i < 10; i--)

Problema:

Si empiezas en 0 y restas (i--), i será -1, -2, -3... siempre será menor a 10. Nunca para.

Solución:

Verifica siempre que tu lógica de actualización (i++ o i--) se acerque a romper la condición, no a alejarse.

Correcto:

for (let i = 0; i < 10; i++)  // ✅ Correcto
for (let i = 10; i > 0; i--)  // ✅ Correcto

El Error "Off-by-one" (Por uno)

Error:

Quieres recorrer 5 veces pero escribes:

for (let i = 0; i <= 5; i++)

Problema:

De 0 a 5 hay 6 números, no 5:

0, 1, 2, 3, 4, 5 = 6 iteraciones ❌

✅ Solución:

Si empiezas en 0, usa <. Si empiezas en 1, usa <=.

// 5 iteraciones correctas (0,1,2,3,4)
for (let i = 0; i < 5; i++)   // ✅

// 5 iteraciones correctas (1,2,3,4,5)
for (let i = 1; i <= 5; i++)  // ✅

Scope de var vs let en Ciclos

❌ Usando var:

for (var i = 0; i < 5; i++) {
  console.log(i);
}
console.log(i); // 5
// ❌ i sigue existiendo!

✅ Usando let:

for (let i = 0; i < 5; i++) {
  console.log(i);
}
console.log(i); // ERROR!
// ✅ i sólo existe en el ciclo

Regla de Oro:

Siempre usa let en el for. Con var, la variable i sigue existiendo fuera del ciclo, lo cual puede causar errores inesperados.

Ejercicio Independiente Aplicado: El "FizzBuzz"

¿Qué es el FizzBuzz?

Este es un ejercicio de programación clásico y legendario que combina:

  • Ciclos (for)
  • Condicionales (if...else if...else)
  • Operador módulo (%)

Instrucciones

1️⃣ Ciclo for del 1 al 100

Escribe un ciclo for que cuente del 1 al 100.

2️⃣ Aplica las reglas para cada número

Si divisible por 3 Y 5 (ej. 15):

Imprime: "FizzBuzz"

SI NO, pero si divisible por 3 (ej. 9):

Imprime: "Fizz"

SI NO, pero si divisible por 5 (ej. 10):

Imprime: "Buzz"

Para todos los demás números:

Imprime el número en sí (ej. 7)

Usa el operador módulo (%)

Para saber si un número es divisible por otro, usa numero % 3 === 0. Significa que "el residuo de numero dividido por 3 es 0", o sea, es divisible.

Criterios de Logro Esperados

  • ☑ El ciclo for se ejecuta 100 veces.
  • ☑ Se utiliza una estructura if...else if...else.
  • ☑ El orden de las condiciones es correcto (la comprobación de "FizzBuzz" debe ir primero).
  • ☑ La salida en consola es correcta: 1, 2, Fizz, 4, Buzz, Fizz, 7, 8, Fizz, Buzz, 11, Fizz, 13, 14, FizzBuzz, ...

Solución Propuesta:

// Ciclo for del 1 al 100
for (let i = 1; i <= 100; i++) {
  
  // Aplicamos las reglas
  if (i % 3 === 0 && i % 5 === 0) {
    // Divisible por 3 Y 5
    console.log("FizzBuzz");
    
  } else if (i % 3 === 0) {
    // Divisible solo por 3
    console.log("Fizz");
    
  } else if (i % 5 === 0) {
    // Divisible solo por 5
    console.log("Buzz");
    
  } else {
    // Todos los demás números
    console.log(i);
  }
}

Salida esperada (primeros 20 elementos):

Fizz
4
Buzz
Fizz
7
8
Fizz
Buzz
11
Fizz
13

FizzBuzz
16

Fizz

Buzz

Conceptos Clave Utilizados:

  • Ciclo for: for (let i = 1; i <= 100; i++) para iterar del 1 al 100
  • Operador módulo (%): i % 3 === 0 verifica divisibilidad
  • Operador lógico AND (&&): i % 3 === 0 && i % 5 === 0 verifica dos condiciones
  • Condicionales: if...else if...else para tomar decisiones
  • Orden de condiciones: Importante verificar "FizzBuzz" primero, sino 15 sería "Fizz" nomás

¿Debería Saber Algo Más?

Ciclos Anidados (Nested Loops)

Puedes poner un for dentro de otro for. Esto es común para recorrer matrices (tablas de Excel con filas y columnas).

Ejemplo: Tabla de Multiplicar (5×5)

// Ciclo externo: filas
for (let i = 1; i <= 5; i++) {
  // Ciclo interno: columnas
  for (let j = 1; j <= 5; j++) {
    console.log(`${i} × ${j} = ${i * j}`);
  }
}

Advertencia: Rendimiento

Si tienes un ciclo de 100 vueltas dentro de otro de 100 vueltas, harás 10,000 operaciones. Esto puede hacer lenta tu aplicación.

Cálculo:

100 ciclos × 100 ciclos = 10,000 operaciones

Concepto: En el futuro aprenderás sobre Big O Notation, que es una forma de medir la eficiencia de tu código.

Iteradores Modernos

En el futuro, cuando veamos Arreglos (Arrays), dejaremos de usar tanto el for clásico y pasaremos a usar versiones más modernas y limpias.

for (clásico)

for (let i = 0; i < arr.length; i++) {
  console.log(arr[i]);
}

for...of (moderno)

for (let item of arr) {
  console.log(item);
}

.forEach() (moderno)

arr.forEach(item => {
  console.log(item);
});

Importante:

Entender el for clásico es obligatorio para entender la lógica base de cómo funcionan los ciclos. Los iteradores modernos son solo "azúcar sintáctico" que hacen el código más limpio, pero por debajo siguen siendo ciclos normales.
